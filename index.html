One system to rule them all

This project implements an automated alerting system in Snowflake to monitor critical events such as:

Task Failures

Out-of-schedule object creation

Failed uploads

Excessive credit usage

Alerts are generated through scheduled queries and procedures, and are automatically sent to Slack or logged in a monitoring table.

Goal: Improve visibility, anticipate incidents and strengthen operational control within the platform.

My Journey

Mapped alert types to system views like QUERY_HISTORY, LOGIN_HISTORY

Created optimized SQL queries with thresholds and filters

Built Slack integration using AWS Lambda and a custom bot

Designed security alert logic to track failed logins by role

Tested end-to-end flows with real data and simulation

Centralized all alerts into ALERT_LOG table
